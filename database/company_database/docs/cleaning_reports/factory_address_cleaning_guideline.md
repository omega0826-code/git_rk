# 공장주소 정제 가이드라인

> 전국공장등록현황 데이터의 공장주소(도로명) 및 공장주소_지번 표준 전처리 절차
> 기존 `address_cleaning_guideline.md`를 공장 데이터용으로 확장

---

## 1. 컬럼 구조

| 컬럼명                   | 설명                               |
| ------------------------ | ---------------------------------- |
| **공장주소_원본**        | 원본 주소 (수정 불가, 이력 보존용) |
| **공장주소_클리닝**      | 전처리 완료된 도로명 주소          |
| **공장주소_지번_원본**   | 원본 지번 주소 (수정 불가)         |
| **공장주소_지번_클리닝** | 전처리 완료된 지번 주소            |

## 2. 전처리 규칙 (순서대로 적용)

### 2-1. 공백 정규화
- 앞뒤 공백 제거
- 연속 공백을 단일 공백으로 치환

### 2-2. 시도 오타 수정
기존 `address_cleaning_guideline.md`의 매핑 테이블 적용

### 2-3. 시도 약어 확장
기존 `address_cleaning_guideline.md`의 약어 테이블 적용

### 2-4. 붙어쓰기 보정 (공장 데이터 특화)

| 패턴           | 보정            |
| -------------- | --------------- |
| `청주시흥덕구` | `청주시 흥덕구` |
| `청주시서원구` | `청주시 서원구` |
| `천안시서북구` | `천안시 서북구` |
| `천안시동남구` | `천안시 동남구` |
| `용인시처인구` | `용인시 처인구` |

### 2-5. 원본 시도명/시군구명 컬럼과 교차 검증
CSV 원본의 `시도명`, `시군구명` 컬럼 값과 정제된 주소의 시도/시군구 비교

## 3. 삭제 조건
- `공장주소`와 `공장주소_지번` **모두 빈칸**인 경우 → 삭제 데이터로 이동
- 시도명만 있고 상세주소 없는 경우 → `[삭제:주소결함]` 태그 부여 후 삭제
