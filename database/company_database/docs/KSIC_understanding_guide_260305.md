> **작성일**: 2026-03-05
> **목적**: 데이터 정제·분석 업무에서 KSIC 업종분류를 활용할 때 필요한 핵심 개념 정리

## 1. KSIC란?

**KSIC**(Korean Standard Industrial Classification, 한국표준산업분류)는 통계청이 관리하는 **산업활동의 표준 분류 체계**입니다. UN의 ISIC(International Standard Industrial Classification)를 기반으로 한국 실정에 맞게 세분화되어 있습니다.

- **관리 기관**: 통계청
- **근거 법령**: 통계법 제22조
- **최신 차수**: **제11차 개정** (2024.01 시행)
- **활용 범위**: 공장등록, 사업자등록, 고용보험, 산업통계 조사 등 모든 공공·민간 산업 통계의 기준

---

## 2. 분류 체계 (5단계 계층 구조)

KSIC는 대분류→세세분류까지 **5단계 트리 구조**로 이루어져 있습니다.

| 단계  | 명칭         | 코드 형식          | 항목 수 (11차) | 예시                         |
| ----- | ------------ | ------------------ | -------------- | ---------------------------- |
| 1단계 | **대분류**   | 알파벳 1자리 (A~U) | 21개           | `C` 제조업                   |
| 2단계 | **중분류**   | 숫자 2자리         | 77개           | `18` 인쇄 및 기록매체 복제업 |
| 3단계 | **소분류**   | 숫자 3자리         | 234개          | `181` 인쇄 및 인쇄관련 산업  |
| 4단계 | **세분류**   | 숫자 4자리         | 501개          | `1811` 인쇄업                |
| 5단계 | **세세분류** | 숫자 5자리         | 1,205개        | `18112` 스크린 인쇄업        |

### 코드 읽기 규칙

```
대표업종코드 예시: 18112 (스크린 인쇄업)

 C        → 대분류: 제조업 (알파벳, 중분류 범위 10~34에 해당)
 18       → 중분류: 인쇄 및 기록매체 복제업 (앞 2자리)
 181      → 소분류: 인쇄 및 인쇄관련 산업 (앞 3자리)
 1811     → 세분류: 인쇄업 (앞 4자리)
 18112    → 세세분류: 스크린 인쇄업 (전체 5자리)
```

> **핵심**: 세세분류 5자리 코드를 알면, 앞자리를 잘라서 상위 분류를 모두 역추적 가능합니다.

## 4. 중분류→대분류 역매핑 방법

대분류 코드는 알파벳이고 세세분류 코드는 숫자이므로, 숫자→알파벳 변환이 필요합니다. **중분류 범위**(위 표의 "중분류 범위" 열)를 이용하여 역매핑 딕셔너리를 구축합니다.

### Python 구현 예시

```python
# KSIC1 대분류 CSV에서 중분류 범위를 파싱하여 매핑 생성
import re, csv

def build_mid_to_major_map(ksic1_path: str) -> dict[str, str]:
    """중분류 코드(2자리) → 대분류 코드(알파벳) 매핑 딕셔너리 생성.

    Args:
        ksic1_path: KSIC1 CSV 파일 경로

    Returns:
        {"10": "C", "11": "C", ..., "99": "U"} 형태의 딕셔너리
    """
    mid_to_major: dict[str, str] = {}
    with open(ksic1_path, "r", encoding="utf-8-sig") as f:
        reader = csv.DictReader(f)
        for row in reader:
            code = row["KSIC1_cd"]            # 예: "C"
            name = row["KSIC1_name"]          # 예: "제조업(10~34)"
            # 괄호 안 숫자 범위 파싱
            match = re.search(r"\((\d+)~?(\d+)?\)", name)
            if match:
                start = int(match.group(1))
                end = int(match.group(2)) if match.group(2) else start
                for mid in range(start, end + 1):
                    mid_to_major[f"{mid:02d}"] = code
    return mid_to_major
```

## 5. 차수(개정 이력)와 변환

### 5.1 주요 개정 차수

| 차수     | 시행 시기 | 비고           |
| -------- | --------- | -------------- |
| 8차      | 2000.01   |                |
| 9차      | 2008.02   |                |
| **10차** | 2017.07   | 11차 직전 버전 |
| **11차** | 2024.01   | **현행 최신**  |

### 5.2 10차→11차 변환 유형

실무 데이터에서 차수가 혼재되어 있을 때, `KSIC10th-KSIC11th.csv` 매핑 파일을 활용합니다.

| 변환 유형    | 건수   | 설명                          | 예시                       |
| ------------ | ------ | ----------------------------- | -------------------------- |
| **(동일)**   | ~1,046 | 코드·명칭 변동 없음           | `01110` → `01110`          |
| **코드변경** | ~75    | 코드 번호만 변경              | `27400` → `27220`          |
| **세분**     | ~60    | 1개 코드가 2개 이상으로 분할  | `30121` → `30121`, `30122` |
| **통합**     | ~36    | 2개 이상이 1개 코드로 합병    | `22111`, `22112` → `22110` |
| **명칭변경** | ~6     | 코드 그대로, 명칭만 변경      | `20112` 명칭 변경          |
| **해설보완** | ~6     | 코드·명칭 그대로, 해설만 변경 | `12000` 해설 보완          |

> **주의 (세분 유형)**: 10차 1개 코드가 11차에서 2개 이상으로 분할된 경우, 매핑 테이블에 **1:N 관계**가 존재합니다. 단순히 첫 번째 매핑만 사용하면 안 되며, 비즈니스 맥락에 따라 적절한 11차 코드를 선택하거나, 대표값을 사용해야 합니다.

## 6. 데이터 정제에서의 실무 활용 패턴

### 6.1 대표업종 코드로 대/중분류 파생

```text
[원본] 대표업종 = 18112, 차수 = 11

① 앞 2자리 추출 → 중분류 코드: "18"
② KSIC2 테이블 조회 → 중분류명: "인쇄 및 기록매체 복제업"
③ 중분류→대분류 역매핑 → 대분류 코드: "C"
④ KSIC1 테이블 조회 → 대분류명: "제조업"
```

### 6.2 차수가 다른 데이터 처리

```text
[원본] 대표업종 = 27400, 차수 = 10

① 매핑 테이블 조회: 27400 (10차) → 27220 (11차) [코드변경]
② 11차 코드 "27220" 기준으로 중분류/대분류 파생
```

### 6.3 결함 판별 체크리스트

| 체크 항목      | 판별 기준                      | 조치                         |
| -------------- | ------------------------------ | ---------------------------- |
| 코드 유효성    | 5자리 숫자가 아님              | 결함 처리                    |
| 분류 존재 여부 | KSIC2 테이블에 앞 2자리가 없음 | 결함 처리                    |
| 차수 불일치    | 10차 코드인데 매핑 불가        | 결함 처리                    |
| 차수 미확인    | 차수가 0/빈칸                  | **11차로 가정** 후 정상 처리 |
