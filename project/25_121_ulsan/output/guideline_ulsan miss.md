## 2. 분석 방법 및 해석 작성 원칙

### 2.1 전체 결과 작성 방식

각 문항에 대해 **전체(n=1,000) 기준**으로 아래 형식에 따라 서술합니다:

```
● [문항 제목]을 살펴본 결과, [1순위 항목]이(가) XX.X%로 가장 많이 나타났으며, 
  이어서 [2순위 항목](XX.X%), [3순위 항목](XX.X%), [4순위 항목](XX.X%) 등의 
  순으로 조사됨
```

**작성 규칙:**
- 상위 3~4개 항목을 비율 내림차순으로 나열
- 비율은 소수점 첫째 자리까지 표시
- 비율이 매우 낮은 항목(5% 미만)은 생략 가능
- 중복응답 문항은 "(중복응답)" 표기
- **문장 끝에 마침표(.)를 사용하지 않음** (예: ~조사됨, ~나타남)

### 2.2 1순위 / 1+2순위 문항 처리 원칙

> [!IMPORTANT]
> 1순위와 1+2순위 문항이 모두 있는 경우, **각각 독립적으로 분석 결과를 작성**합니다.
> - **1순위 문항**: 1순위 데이터에 기반한 해석을 작성
> - **1+2순위 문항**: 1+2순위 데이터에 기반한 해석을 작성 (1순위와 비교하지 않음)
> 
> 즉, 각 문항은 자체 데이터만으로 독립적인 결과 해석을 수행합니다.

**해당 문항:**

| 1순위 문항 | 1+2순위 문항 | 주제              |
| :--------: | :----------: | :---------------- |
|    5번     |     6번      | 취업 정보 세부    |
|    7번     |     8번      | 부족 역량         |
|    9번     |     10번     | 취업 시 고려 요소 |
|    18번    |     19번     | 청년 정착 정책    |

**작성 예시:**
```
### 3-4-1. 필요 취업 정보 (1순위)
● [1순위 데이터 기반 해석]
| 통계표 ... |

### 3-4-2. 필요 취업 정보 (1순위+2순위)
● [1+2순위 데이터 기반 독립 해석]
| 통계표 ... |
```

### 2.3 해석 배치 원칙

> [!IMPORTANT]
> 각 문항의 해석(분석 결과 서술)은 **해당 통계표 바로 위에** 배치합니다.

```markdown
### 3-1. 일자리 정보 탐색 경로

● 일자리 정보 탐색 활용 경로를 살펴본 결과, ...
- 소속대학별로는 ...

| 구 분 | 세부 항목 |  ... |
| :---- | :-------- | ---: |
| ...   | ...       |  ... |
```

### 2.4 분석 변수 선택 규칙

> [!IMPORTANT]
> 보고서에 포함하는 분석 변수는 다음과 같이 구성함
>
> | 구분 | 변수 | 포함 여부 |
> |:---|:---|:---:|
> | **전체** | 전체(n=1,000) | ✅ 포함 |
> | **교차분석 1** | 소속대학별 | ✅ 포함 |
> | **교차분석 2** | 학년별 | ✅ 포함 |
> | **교차분석 3** | 전공별 | ✅ 포함 |
>
> 교차분석 서술 순서: 소속대학별 → 학년별 → 전공별
> 모든 문항에서 3개 변수 모두에 대해 특징적인 내용을 기술함

### 2.5 교차분석(소속대학/학년/전공) 작성 방식

전체 결과 아래에 **특징적인 내용만 간략하게** 기술함:

```
- 소속대학별로는 [대학명]에서 [항목]이 XX.X%로 특히 높게/낮게 나타남
- 학년별로는 [학년]에서 [항목]이 XX.X%로 [특징]
- 전공별로는 [전공]에서 [항목]에 대한 응답이 XX.X%로 두드러짐
```

**작성 규칙:**
- **±10%p 이상 차이**가 나는 특징적인 항목 위주로 기술
- 각 교차변수(소속대학/학년/전공)당 1~2개 핵심 포인트만 추출
- 모든 세부 항목을 나열하지 않고, 정책적 시사점이 있는 내용 중심

> [!IMPORTANT]
> 교차분석(하위특성) 서술 시 **"전체 평균 대비", "전체 대비"** 등의 비교 표현을 사용하지 않음
> 해당 하위 그룹의 수치와 특징만으로 **독립적으로 서술**함
>
> - ❌ 울산과학기술원에서 SNS 활용률이 2.2%로 **전체 평균(21.3%) 대비** 현저히 낮게 나타남
> - ✅ 울산과학기술원에서 SNS 활용률이 2.2%로 현저히 낮게 나타남

## 5. 산출물 관리 규칙

### 5.1 폴더 구조

```
output/
├── report_ulsan miss.md                       ← 가이드라인 (현재 파일)
├── (OUTPUT)울산 일자리 미스매치_260205_1822.xlsx ← 원본 데이터
├── (OUTPUT)울산 일자리 미스매치_260205_1822.md   ← 마크다운 변환 데이터
│
├── scripts/                                   ← 스크립트 (분석·보고서 생성)
│   ├── analyze_data_YYMMDD_HHMM.py
│   └── generate_report_YYMMDD_HHMM.py
│
├── tables/                                    ← 통계표 (개별 문항별)
│   ├── table_02_정보탐색경로_YYMMDD_HHMM.md
│   └── ...
│
├── wording/                                   ← 보고서 워딩 (해석 텍스트)
│   ├── wording_전체_YYMMDD_HHMM.md
│   └── wording_전체_YYMMDD_HHMM.txt
│
└── report/                                    ← 최종 보고서
    └── 울산_일자리_미스매치_분석보고서_YYMMDD_HHMM.md
```

### 5.2 파일 명명 규칙

> [!IMPORTANT]
> 모든 생성 산출물 파일명 끝에는 **타임스탬프**를 포함합니다.
> 
> **형식**: `_YYMMDD_HHMM`
> 
> **예시**: `analyze_data_260209_0930.py`, `wording_전체_260209_0930.md`

| 산출물 유형           | 저장 위치  | 파일명 패턴                                      |
| :-------------------- | :--------- | :----------------------------------------------- |
| 분석 스크립트         | `scripts/` | `analyze_[주제]_YYMMDD_HHMM.py`                  |
| 보고서 생성 스크립트  | `scripts/` | `generate_report_YYMMDD_HHMM.py`                 |
| 통계표                | `tables/`  | `table_[번호]_[주제]_YYMMDD_HHMM.md`             |
| **보고서 워딩 (MD)**  | `wording/` | `wording_[범위]_YYMMDD_HHMM.md`                  |
| **보고서 워딩 (TXT)** | `wording/` | `wording_[범위]_YYMMDD_HHMM.txt`                 |
| 최종 보고서           | `report/`  | `울산_일자리_미스매치_분석보고서_YYMMDD_HHMM.md` |

### 5.3 보고서 워딩 저장 규칙

> [!NOTE]
> 보고서에 사용되는 해석 텍스트(워딩)는 **MD 파일과 TXT 파일 두 가지 형식**으로 별도 저장합니다.
> - **MD 파일**: 마크다운 서식 포함 (볼드, 목록 등)
> - **TXT 파일**: 순수 텍스트만 포함 (서식 없이 복사·붙여넣기 용도)
