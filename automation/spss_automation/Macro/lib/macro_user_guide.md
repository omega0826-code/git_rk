# SPSS 공통 매크로 라이브러리 — 사용자 가이드

> **Version**: 1.0.0 | **최종 수정**: 2026-02-26

---

## 1. 개요

프로젝트별 SPSS syntax 작성 시 반복 사용되는 **43개 분석 매크로**를 라이브러리로 관리합니다.

| 사용 방식                | 설명                                             | 적합한 경우                |
| ------------------------ | ------------------------------------------------ | -------------------------- |
| **방식 A** 복사 붙여넣기 | `common_macros.sps`를 syntax에 복사 후 배너 수정 | 빠르게 시작, 단일 프로젝트 |
| **방식 B** 자동 생성     | JSON 설정 → Python 스크립트 → 맞춤 매크로 생성   | 여러 프로젝트 관리, 자동화 |

---

## 2. 방식 A — 복사 붙여넣기

### Step 1: 매크로 복사

`Macro/lib/common_macros.sps` 파일의 전체 내용을 프로젝트 syntax 파일 상단에 복사합니다.

### Step 2: 배너 변수 수정

편집기의 **찾기/바꾸기** 기능으로 아래 문자열을 프로젝트 배너로 교체:

| 찾기 (현재 배너)                          | 바꾸기 (프로젝트 배너 예시) |
| ----------------------------------------- | --------------------------- |
| `t1+a3+a6r+a7r+a8+a9+a10r+a11_1r+ a11_2r` | `t1+sq1+sq2+sq3+q1r`        |
| `a3 a6r a7r a8 a9 a10r a11_1r  a11_2r`    | `sq1 sq2 sq3 q1r`           |

> **주의**: 공백 수까지 정확히 일치해야 합니다. 원본의 공백 패턴을 확인하세요.

### Step 3: 매크로 사용

```sps
freqt a='B1. 공동활용 연구장비를 활용하십니까?' / x=b1.
```

## 3. 방식 B — 자동 생성

### Step 1: JSON 설정 파일 작성

`banner_config_sample.json`을 복사하여 프로젝트에 맞게 수정:

```json
{
  "project_name": "내 프로젝트",
  "banner": "t1+sq1+sq2+sq3+q1r",
  "banner_list": "sq1 sq2 sq3 q1r",
  "banner_no_t1": "sq1+sq2+sq3+q1r",
  "ptotal1": "전 체",
  "ptotal2": "구 분",
  "validn_label": "응답수"
}
```

### Step 2: 매크로 파일 생성

```bash
python generate_macros.py my_config.json -o my_project_macros.sps
```

### Step 3: SPSS에서 사용

```sps
* INCLUDE로 로드하거나 내용을 복사
INCLUDE 'my_project_macros.sps'.

freqt a='Q1. 만족도' / x=q1.
```

### CLI 옵션

| 인자             | 필수 | 기본값                           | 설명                |
| ---------------- | ---- | -------------------------------- | ------------------- |
| `config`         | ✅    | —                                | 배너 설정 JSON 경로 |
| `-o`, `--output` | ✅    | —                                | 출력 SPS 경로       |
| `--template`     | —    | `lib/common_macros_template.sps` | 커스텀 템플릿 경로  |
| `--encoding`     | —    | `utf-8`                          | 출력 인코딩         |

## 4. 배너 설정 가이드

### JSON 키 설명

| 키             | 설명              | 규칙                     |
| -------------- | ----------------- | ------------------------ |
| `banner`       | TABLE 행 변수     | `t1`로 시작, `+`로 구분  |
| `banner_list`  | 통계량 변수       | `t1` 제외, 공백으로 구분 |
| `banner_no_t1` | 행 변수 (t1 제외) | `+`로 구분               |
| `ptotal1`      | 전체 소계 라벨    | 보통 `전 체`             |
| `ptotal2`      | 구분 소계 라벨    | 보통 `구 분`             |
| `validn_label` | 유효 응답 라벨    | `업체수`, `응답수` 등    |

### 프로젝트별 설정 예시

```json
// 기업 조사
{"banner": "t1+a3+a6r+a7r+a8", "banner_list": "a3 a6r a7r a8", "validn_label": "업체수"}

// 주민 만족도 조사
{"banner": "t1+sq1+sq2+sq3", "banner_list": "sq1 sq2 sq3", "validn_label": "응답수"}
```

## 5. 매크로 활용 예시

### 객관식 문항

```sps
* 단순 빈도 + 비율 + 제목
freqt a='Q1. 귀하의 성별은?' / x=q1.

* 5점 척도 문항 (빈도 + 평균)
freqm2t a='Q2. 만족도' / x=q2 y=q2r.
```

### 주관식/수치 문항

```sps
* 평균 (소수점 1자리) + 제목
m1t a='현황' / x=a6 a7.

* 업체수 + 평균 + 합계
cms1t a='예산 현황' / x=a6 a7.
```

### 복수응답 문항

```sps
* 빈도 + 비율 + 제목
PRT b='Q5. 이용 목적(중복)' / x=q5_1 y=q5_5 z=aa a='이용 목적(중복)'.
```

## 6. 인코딩 가이드

| 환경                     | 권장 인코딩 | 방법                              |
| ------------------------ | ----------- | --------------------------------- |
| SPSS 25.0 한국어 Windows | CP949       | `--encoding cp949`                |
| SPSS 28+ / Unicode       | UTF-8       | `--encoding utf-8` (기본)         |
| 복사 붙여넣기 (방식 A)   | —           | SPSS 버전에 맞게 파일 인코딩 설정 |

> **🔥 CP949(EUC-KR) 사용 시 주의사항**
> 한글 Windows SPSS 25.0 이하 버전에서는 CP949를 사용해야 한글이 깨지지 않습니다.
> 하지만 워드나 웹에서 복사한 **특수기호**(예: `═`, `·`, `–`, 스마트 따옴표 등)는 CP949에서 지원하지 않아 저장/변환 시 오류(`UnicodeEncodeError`)가 발생할 수 있습니다.
> `generate_macros.py`와 `md_to_sps.py`는 이를 자동으로 ASCII 문자로 대체(예: `═` → `=`)하여 오류를 방지합니다.

## 7. 매크로 확장

새 매크로 추가 시 **3개 파일 동시 수정**:

1. `common_macros.sps` — DEFINE 추가
2. `common_macros_template.sps` — 배너를 `{{BANNER}}`로 치환하여 추가
3. `common_macros_reference.md` — 설명·파라미터·예시 추가

변경 후 `CHANGELOG.md`에 이력 기록.

## 8. FAQ

### Q: INCLUDE로 로드하면 매크로가 인식되지 않아요
- INCLUDE 경로에 한글이 포함되면 오류 발생 가능 → 영문 경로 사용 권장
- INCLUDE 경로는 **작은따옴표**로 감싸야 합니다

### Q: 배너 변수 교체 후 일부 매크로가 오류 나요
- 찾기/바꾸기 시 **공백 수**가 원본과 정확히 일치하는지 확인
- 방식 B(자동 생성)를 사용하면 이 문제를 방지할 수 있습니다

### Q: 실행 시 매크로 정의 관련 알 수 없는 문법 오류가 나요
- **원인**: SPSS는 `DEFINE ... !ENDDEFINE` 블록 내부에 **빈 줄(Blank Line)**이 있으면 그곳을 매크로 정의의 끝 또는 명령의 경계로 잘못 인식하여 파싱 오류를 일으킵니다.
- **해결**: 매크로 템플릿이나 소스 코드의 `DEFINE` 블록 안에 빈 줄이 들어가지 않도록 주의하세요. (제공된 파이썬 스크립트에는 자동 제거 기능이 포함되어 있습니다.)

### Q: 기존 프로젝트 매크로와 호환되나요
- 매크로 이름과 파라미터가 동일하므로 **100% 호환**됩니다
- 실행부(매크로 호출 부분)는 수정 없이 그대로 사용 가능
