# SPSS 자동화 도구 트러블슈팅 DB

> 발생한 모든 오류를 축적하여 향후 동일 문제 재발을 방지합니다.

---

## TS-001: CP949 UnicodeEncodeError

| 항목          | 내용                                                                  |
| ------------- | --------------------------------------------------------------------- |
| **증상**      | `UnicodeEncodeError: 'cp949' codec can't encode character '\u2550'`   |
| **원인**      | 마크다운의 이중선(`═`), 가운뎃점(`·`), en dash(`–`) 등은 CP949에 없음 |
| **해결**      | `CP949_REPLACE_MAP` 딕셔너리로 ASCII 문자로 자동 대체                 |
| **영향 버전** | v1.0.0에서 발견, v1.0.0에서 수정                                      |
| **회귀 방지** | CP949 출력 시 항상 대체 맵 적용. 신규 특수문자 발견 시 맵에 추가      |

---

## TS-002: DEFINE 내 빈 줄로 SPSS 파싱 오류

| 항목          | 내용                                                                               |
| ------------- | ---------------------------------------------------------------------------------- |
| **증상**      | SPSS에서 매크로 실행 시 알 수 없는 문법 오류 (Error 120)                           |
| **원인**      | SPSS는 `DEFINE...!ENDDEFINE` 안의 빈 줄을 명령 경계로 인식                         |
| **해결**      | `md_to_sps.py`에 `--strip-define-blanks` 옵션, `generate_syntax.py`는 빈 줄 미생성 |
| **영향 버전** | v1.0.0에서 발견, v1.0.0에서 수정                                                   |
| **회귀 방지** | 매크로 생성 시 DEFINE 블록 내에 빈 줄을 넣지 않도록 설계                           |

## TS-003: 왕복 변환 줄바꿈 미보존

| 항목          | 내용                                                              |
| ------------- | ----------------------------------------------------------------- |
| **증상**      | SPS→MD→SPS 변환 시 파일 크기 불일치 (CRLF→LF)                     |
| **원인**      | Python 텍스트 모드(`"r"`)가 `\r\n`을 `\n`으로 자동 변환           |
| **해결**      | 바이너리 모드(`"rb"`, `"wb"`) 사용 + 메타 정보에 줄바꿈 패턴 기록 |
| **영향 버전** | v1.0.0                                                            |
| **회귀 방지** | 파일 I/O는 항상 바이너리 모드 기본                                |

---

## TS-004: strip() 함수로 원본 빈 줄 손실

| 항목          | 내용                                        |
| ------------- | ------------------------------------------- |
| **증상**      | 파일 시작/끝의 빈 줄이 사라짐               |
| **원인**      | `strip("\n")`이 모든 앞뒤 줄바꿈을 제거     |
| **해결**      | strip 제거, 정규식으로 정확한 경계만 매칭   |
| **영향 버전** | v1.0.0                                      |
| **회귀 방지** | 원본 보존이 필요한 곳에서는 strip 사용 금지 |

---

## TS-005: SPS 파일 0바이트 생성

| 항목          | 내용                                                                         |
| ------------- | ---------------------------------------------------------------------------- |
| **증상**      | 인코딩 변환 스크립트 실행 후 SPS 파일이 0바이트                              |
| **원인**      | `f.write()` 전에 인코딩 에러로 예외 발생, `"wb"` 모드가 파일을 이미 truncate |
| **해결**      | 인코딩 변환 전에 CP949 호환 불가 문자를 먼저 대체                            |
| **영향 버전** | v1.0.0에서 발견, v1.0.0에서 수정                                             |
| **회귀 방지** | `generate_syntax.py`는 인코딩 대체를 내장하여 별도 변환 불필요               |
