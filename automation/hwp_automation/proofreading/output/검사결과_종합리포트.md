## 6. 페이지 매핑 구현 과정 및 원인 분석

### 6-1. 문제 발생 이력

| 시도 | 방식                                            | 결과                               | 원인                                                         |
| ---- | ----------------------------------------------- | ---------------------------------- | ------------------------------------------------------------ |
| 1차  | `MovePos(8)` — 다음 문단으로 이동               | **1개 문단만 매핑**                | `MovePos(8)`이 즉시 False 반환 (문서 구조에따라 동작 불안정) |
| 2차  | `MoveParaEnd + MoveRight` + `KeyIndicator()[0]` | **1,151개 매핑, 모두 True**        | 인덱스 0은 성공 여부(bool), 페이지 번호는 인덱스 1           |
| 3차  | `KeyIndicator()[1]` 수정                        | **1,151개 매핑, 모두 1페이지**     | `visible=False`에서 페이지 레이아웃 미계산                   |
| 4차  | **`goto_page()` 역방향 매핑**                   | **1,147개 매핑, 168페이지 정확** ✅ | —                                                            |

### 6-2. 근본 원인

**pyhwpx `visible=False` 모드의 한계:**

| API                     | `visible=True` | `visible=False`            |
| ----------------------- | -------------- | -------------------------- |
| `PageCount`             | ✅ 정확         | ✅ 정확                     |
| `KeyIndicator()` 페이지 | ✅ 정확         | ❌ 항상 1 (레이아웃 미계산) |
| `goto_page()`           | ✅ 정확         | ✅ 정확                     |
| `current_page`          | ✅ 정확         | ✅ 정확 (goto_page 후)      |
| `GetPos()`              | ✅ 정확         | ✅ 정확                     |

- 한글은 `visible=False`에서 화면 렌더링을 하지 않아 `KeyIndicator()`가 정확한 페이지를 계산하지 못함
- 그러나 `goto_page()`는 내부적으로 페이지 레이아웃 정보를 활용하므로 정확하게 동작
- `GetPos()` 반환값의 `[1]`은 0-based 문단 인덱스이므로 매핑에 활용 가능

### 6-3. 최종 해결 방식 (향후 작업에서도 동일하게 사용)

```python
# 각 페이지로 이동하여 시작 문단 위치 수집 (역방향 매핑)
page_start_paras = []
for page in range(1, total_pages + 1):
    hwp.goto_page(page)
    pos = hwp.GetPos()          # (list_id, para_id, char_pos)
    page_start_paras.append((page, pos[1]))  # pos[1]은 0-based 문단 인덱스

# 범위 기반 매핑: 각 페이지 시작 ~ 다음 페이지 시작 전까지
page_start_paras.sort(key=lambda x: x[1])
for i in range(len(page_start_paras)):
    page_num, start = page_start_paras[i]
    end = page_start_paras[i+1][1] if i+1 < len(page_start_paras) else total_paras
    for para in range(start, end):
        page_map[para + 1] = page_num  # 1-based
```

### 6-4. 주의사항 (다음 작업 시 참고)

1. **`visible=False`에서 `KeyIndicator()` 사용 금지** — 항상 1페이지로 반환됨
2. **`goto_page()` + `GetPos()` 조합** 사용 — 가장 안정적
3. **`MovePos(8)` 미사용** — 문서 구조에 따라 동작이 불안정
4. **문단 인덱스 불일치 가능** — pyhwpx의 문단 수(표 내부 포함 1,147개)와 hwpx_parser의 본문 문단 수(948개)가 다름. 페이지 매핑은 pyhwpx 기준으로 수행하되, 검사 결과의 줄 번호(hwpx_parser 기준)와의 매핑에서 근사값 사용
5. **XML `pageBreak` 폴백** — pyhwpx가 없는 환경에서는 `_fallback_xml_page_map()` 사용. 정확도는 낮으나 대략적 위치 제공
