## 3. 오류 수정 이력

### 3-1. CMD 경로 공백 문제

**증상**: `cmd.exe`에서 공백이 포함된 경로(`Hwp Automation`)를 직접 인수로 전달하면 경로가 분리됨.

```
python: can't open file 'd:\\git_rk\\"d:\\git_rk\\Hwp': [Errno 22] Invalid argument
```

**해결**: 래퍼 스크립트(`_run_typo.py`)를 작성하여 `subprocess.run()`으로 경로를 안전하게 전달.

```python
result = subprocess.run(
    [sys.executable, script, hwpx_file],
    cwd=r"d:\git_rk", env=env,
)
```

### 3-2. CP949 인코딩 오류

**증상**: em dash(`—`) 문자가 Windows CP949 코드페이지에서 인코딩 불가.

```
UnicodeEncodeError: 'cp949' codec can't encode character '\u2014'
```

**해결**: 래퍼 스크립트에서 `PYTHONIOENCODING=utf-8` 환경 변수를 강제 설정.

```python
env = os.environ.copy()
env['PYTHONIOENCODING'] = 'utf-8'
```

또한 스크립트 내 em dash(`—`)를 ASCII 호환 문자(`--`)로 대체.

## 4. 기능 업그레이드 (V2.0)

### 4-1. 페이지 매핑 기능 추가

**목표**: 각 이슈의 원본 문서 페이지 번호를 리포트에 표시

#### 시도 1: `KeyIndicator()` 방식 (실패)

```python
page_num = self.hwp.KeyIndicator()[0]  # → True (boolean) 반환
```

- `KeyIndicator()`의 인덱스 0은 성공 여부(bool)였음
- 인덱스 1이 페이지 번호임을 확인하여 수정

#### 시도 2: `KeyIndicator()[1]` 방식 (부분 실패)

```python
page_num = key_info[1]  # → 항상 1 반환
```

- `visible=False` 모드에서 한글이 **페이지 레이아웃 렌더링을 수행하지 않아** 페이지가 항상 1로 반환됨
- 1,151개 문단 전체가 1페이지로 매핑되는 문제 발생

#### 시도 3: `goto_page()` 역방향 매핑 (성공 ✅)

```python
# 각 페이지로 이동하여 시작 문단 위치 수집
for page in range(1, self.total_pages + 1):
    self.hwp.goto_page(page)
    pos = self.hwp.GetPos()
    page_start_paras.append((page, pos[1]))

# 범위 기반 매핑: 각 페이지 시작~다음 페이지 시작
for i in range(len(page_start_paras)):
    page_num, start_para = page_start_paras[i]
    end_para = page_start_paras[i+1][1] if i+1 < len(...) else total
    for para_0 in range(start_para, end_para):
        self._page_map[para_0 + 1] = page_num
```

- **168페이지, 1,147개 문단** 성공 매핑
- `goto_page()`는 `visible=False`에서도 정확한 위치 반환 확인
- XML `pageBreak` 폴백도 유지 (pyhwpx 미설치 환경 대응)
