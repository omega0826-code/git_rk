## 3. 개별 오류 상세

### 오류 #1: Python 시작 지연

**오류 설명**
- `python hwpx_parser.py` 실행 시 Python 프로세스 자체가 수 분간 시작되지 않음
- `python --version`도 5분 이상 응답 없는 심각한 지연

**재현 조건**
- Windows 10 환경
- 에이전트 터미널에서 실행 시 발생
- 사용자 직접 실행 시에는 정상 (2.2초 완료)

**원인 분석**
- Windows Defender의 실시간 스캔이 새 Python 프로세스 시작을 지연시킴
- 에이전트의 cmd 터미널 환경과 사용자의 가상환경(`.venv`) 차이 가능성
- 이전 대화에서도 동일 증상 발견 (Python 실행 지연 해결 세션 참조)

**해결 방법**
1. 사용자가 직접 터미널에서 실행 → 정상 작동 확인
2. Windows Defender에서 Python 경로 및 프로젝트 디렉토리를 제외 목록에 추가
3. `PYTHONUNBUFFERED=1` 환경변수 설정 (출력 버퍼링 방지)

**회피 전략**
- 파서 실행은 사용자 터미널에서 직접 실행 권장
- `.venv` 가상환경 활성화 후 실행

---

### 오류 #2: 표 제목 추측 실패

**오류 설명**
- `TableExtractor._guess_table_title()`이 일부 표에서 빈 문자열 반환
- 결과적으로 CSV 파일명이 `table_NN.csv`로만 저장됨 (제목 없음)

**재현 조건**
- 표 바로 위에 문단이 없는 경우
- 표가 `hp:p` 요소 내부에 직접 포함되지 않는 경우
- 표 앞에 이미지나 다른 객체가 있는 경우

**원인 분석**
- 표 제목 추측 로직이 `tbl.getparent()` → 이전 형제 `hp:p`를 탐색하는 방식
- HWPX XML 구조에서 표가 문단 내부에 포함되는 방식이 일정하지 않음

**해결 방법 (현재)**
- 제목이 없는 경우 `table_NN.csv`로 저장 (기능적 문제 없음)
- 사용자가 수동으로 파일명 변경 가능

**향후 개선 방안**
- `hp:caption` 요소를 활용한 표 캡션 추출 기능 추가
- 표의 첫 행(헤더)을 기반으로 제목 추측 보강

---

### 오류 #3: styleIDRef=0 레벨 매핑 모호

**오류 설명**
- 원문자(①②③) 번호가 있는 소제목의 `styleIDRef`가 `0`
- `STYLE_LEVEL_MAP`에서 0은 매핑되지 않아 기본값으로 처리됨
- 본문 텍스트와 구분이 어려울 수 있음

**재현 조건**
- `hp:titleMark`가 있지만 `styleIDRef="0"`인 문단

**원인 분석**
- HWPX에서 `styleIDRef=0`은 "본문" 스타일을 의미
- 그러나 `hp:titleMark`가 있으므로 실제로는 소제목 역할
- 한컴오피스에서 개요 번호를 사용하지 않고 원문자를 직접 입력한 것으로 추정

**해결 방법**
- `HeadingDetector`에서 `titleMark` 존재 시 무조건 제목으로 분류
- level은 0으로 설정되어 별도 처리 가능

**향후 개선 방안**
- `styleIDRef=0` + `titleMark` 조합을 별도 레벨(예: 4)로 매핑
- `header.xml`의 스타일 정의를 참조하여 실제 글꼴 크기 기반 레벨 결정
