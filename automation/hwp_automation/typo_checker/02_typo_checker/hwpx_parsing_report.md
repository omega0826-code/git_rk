## 6. 오타 검사용 활용 방법

### 텍스트 오타 검사
- **`full_text.md`** 또는 **`full_text.txt`**: 전체 본문 텍스트를 대상으로 맞춤법/오타 검사 가능
- 문단 단위로 분리되어 있어 위치 추적 용이

### 표 데이터 검사
- **개별 CSV 파일**: 표별로 분리되어 숫자/텍스트 오류 확인 가능
- **`all_tables.xlsx`**: Excel에서 시트별로 표를 열어 데이터 일관성 검증
- 주요 검사 항목: 합계 일치 여부, 단위 표기 통일, 연도 순서 등

### 검사 우선순위 제안
1. `full_text.txt` 전체 맞춤법 검사
2. 대형 표(표 24, 34, 38, 40)의 숫자 합계 검증
3. 표/본문 간 숫자 일치 여부 교차 검증

## 7. 특이사항

1. **문서 범위**: 파싱된 텍스트는 주로 "비용 추정" ~ "재무성 분석" 파트임 (I~V장 목차 포함, VI장 타당성분석 본문 중심)
2. **표 제목 미추출**: 원본 문서에서 표 제목이 `< 표제목 >` 형식이나, 파서의 제목 추측 로직(`_guess_table_title`)이 빈 문자열 반환 → CSV 파일명에 제목 미반영
3. **수식/특수기호**: 편익/비용 수식의 변수명과 특수기호(Σ 등)가 일부 텍스트로 추출되지 않을 수 있음
4. **이미지 설명**: image_01은 시그니처 로고(`3.시그니처 가로형 국영문.jpg` 원본), image_02는 미기재
